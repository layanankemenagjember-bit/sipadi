⚠️ Panduan teknis untuk developer. Pastikan memiliki akun Cloudflare (gratis) — diperlukan kartu kredit/debit untuk aktivasi R2 (billing Rp 0 jika di bawah kuota gratis).

1 Buat Bucket R2 di Cloudflare

aLogin ke dash.cloudflare.com → pilih akun Anda → klik R2 Object Storage di sidebar kiri
bKlik "Create bucket" → nama bucket: sipadi-arsip → Region: APAC → Create
cDi tab bucket → klik "Settings" → cari Public Access → klik "Allow Access" → salin URL pub-xxxx.r2.dev

2 Buat API Token R2

aDi halaman R2 → klik "Manage R2 API tokens" (pojok kanan atas)
bKlik "Create API token" → beri nama: sipadi-upload → Permission: Admin Read & Write → Create
cSalin Access Key ID dan Secret Access Key — hanya tampil sekali, simpan di tempat aman!
dSalin Account ID dari halaman R2 utama (pojok kanan atas dashboard Cloudflare)

3 Konfigurasi CORS Bucket R2

// R2 Bucket → Settings → CORS Policy → Add CORS Policy
// Paste JSON berikut:
[
  {
    "AllowedOrigins": ["*"],
    "AllowedMethods": ["GET", "PUT", "HEAD"],
    "AllowedHeaders": ["*"],
    "ExposeHeaders":  ["ETag"],
    "MaxAgeSeconds":  3600
  }
]
// PENTING: Ganti "*" dengan domain resmi saat production
// Contoh: ["https://sipadi.kemenagjember.id"]

4 Isi Konfigurasi di File SIPADI

// Di bagian KONFIGURASI di file sipadi.html — isi nilai berikut:
const R2_ACCOUNT_ID  = 'a1b2c3d4e5f6...';          // dari Cloudflare Dashboard
const R2_ACCESS_KEY  = 'abc123...';                  // Access Key ID dari token R2
const R2_SECRET_KEY  = 'xyz789...';                  // Secret Access Key dari token R2
const R2_BUCKET      = 'sipadi-arsip';               // nama bucket yang dibuat di step 1
const R2_PUBLIC_URL  = 'https://pub-xxxx.r2.dev';   // URL publik bucket (dari step 1c)

// Upload akan menggunakan AWS Signature V4 — sudah diimplementasi di fungsi uploadFile()
// Tidak perlu install library AWS SDK tambahan

5 Verifikasi Upload

// Test upload dari Console browser (F12):
const testFile = new File(["test content"], "test.txt", {type:"text/plain"});
uploadFile(testFile, "nikah/test").then(url => {
  console.log("✅ Upload berhasil! URL:", url);
  // URL format: https://pub-xxxx.r2.dev/nikah/test/[timestamp]-[random].txt
}).catch(e => console.error("❌ Upload gagal:", e));

// Atau upload file asli via modal Register Nikah / Arsip Ijazah
// Cek tab Network di DevTools → filter "r2.cloudflarestorage.com"
// Response 200 = berhasil

💡 Kuota Gratis R2 (per bulan)

📦 Storage: 10 GB gratis — cukup untuk ±17.000 berkas nikah/tahun dengan kompresi PDF
📤 Operasi Class A (PUT/upload): 1 juta request gratis per bulan
📥 Operasi Class B (GET/download): 10 juta request gratis per bulan
🚀 Egress (bandwidth keluar): GRATIS tanpa batas — keunggulan utama R2 vs S3/GCS

OPSI A — Deploy Drag & Drop (Termudah)

REKOMENDASI
1
Buat akun Netlify

Buka netlify.com → klik Sign up → pilih "Continue with GitHub" atau Google

2
Buka halaman Sites

Setelah login, buka dashboard → klik tab "Sites" → scroll ke bawah

3
Drag & drop folder project

Seret folder yang berisi arsip-kemenag-jember.html (rename jadi index.html) ke area drop di halaman Sites Netlify

4
Selesai! URL langsung aktif

Netlify akan memberikan URL seperti https://random-name-123.netlify.app

⚡

OPSI B — Deploy via Netlify CLI (Profesional)

ADVANCED
# 1. Install Netlify CLI secara global
npm install -g netlify-cli

# 2. Login ke akun Netlify
netlify login
# Browser akan terbuka → otorisasi CLI

# 3. Masuk ke folder project
cd /path/to/sipadi-project

# 4. Inisialisasi project Netlify
netlify init
# Pilih: "Create & configure a new site"
# Nama site: sipadi-kemenag-jember (atau kosongkan untuk auto)

# 5. Deploy ke production
netlify deploy --prod --dir=.
# --dir=.  jika file HTML ada di root folder
# --dir=dist  jika menggunakan build (React/Vite)

# ✅ URL production akan muncul:
# https://sipadi-kemenag-jember.netlify.app

📄 File netlify.toml (Konfigurasi Lanjutan)

# netlify.toml — letakkan di root folder project

[build]
  publish = "dist"           # folder output build
  command = "npm run build"  # perintah build

[build.environment]
  NODE_VERSION = "18"

# Redirect untuk SPA (React Router)
[[redirects]]
  from   = "/*"
  to     = "/index.html"
  status = 200

# Custom headers untuk keamanan
[[headers]]
  for = "/*"
  [headers.values]
    X-Frame-Options        = "DENY"
    X-XSS-Protection       = "1; mode=block"
    X-Content-Type-Options = "nosniff"
    Referrer-Policy        = "strict-origin-when-cross-origin"
    Content-Security-Policy = "default-src 'self'; script-src 'self' 'unsafe-inline' https://*.firebaseapp.com https://*.googleapis.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src 'self' https://fonts.gstatic.com; connect-src 'self' https://*.googleapis.com https://*.firebaseio.com wss://*.firebaseio.com;"

🔐 Domain Kustom (Opsional)

1
Netlify → Site Settings → Domain Management

Klik "Add custom domain" → masukkan domain, misal: sipadi.kemenag-jember.go.id

2
Update DNS Record

Di panel domain Anda, tambahkan CNAME record: sipadi → [nama-site].netlify.app

3
SSL Otomatis

Netlify akan otomatis menerbitkan sertifikat SSL gratis via Let's Encrypt dalam 2–5 menit

✅ Checklist Sebelum Go-Live

  • File index.html ada di root atau folder publish yang dikonfigurasi
  • Firebase Config menggunakan environment variables (bukan hardcode)
  • Authorized domains di Firebase Auth sudah ditambahkan domain Netlify
  • Security Rules Firestore sudah dipublish (bukan test mode)
  • Storage Rules sudah dikonfigurasi
  • netlify.toml dikonfigurasi (terutama redirects untuk SPA)
  • Environment variables semua terisi di Netlify dashboard
  • Uji coba login Google dari domain Netlify (bukan localhost)
  • Verifikasi operasi CRUD dari domain production
  • Backup data Firestore awal sudah diisi via Firebase Console
